<Update label="June 2026" description="Hacktron Platform">
  ## GitLab parity, enterprise GitHub, and smarter triage

  This release brings GitLab reviews to parity with GitHub, adds support for
  self-hosted GitHub, and keeps triage conversations in the tools your team
  already uses.

  **GitHub Enterprise Server** — You can now connect a self-hosted GitHub
  Enterprise Server instance and run both white-box scans and PR reviews
  against it, scoped cleanly to each connection and host.

  **GitLab merge request reviews** — Merge request comments and finding triage
  are now on par with GitHub. Trigger a review with `@hacktronai review`
  directly on a merge request, and Hacktron posts a findings summary before its
  inline discussions.

  **Triage that follows the conversation** — Finding triage decisions now appear
  directly in the originating GitHub PR or GitLab MR thread, so the discussion
  stays in one place.

  **Pull request triage in the review flow** — Finding triage is now built into
  the review flow, so you can act on findings without leaving the PR.

  **Pentest commit targeting** — When starting a pentest you can now pick a
  specific tag or commit SHA in the repository step.

  **Org-pooled billing and a Usage page** — Code Review billing is now pooled
  across your organization, with annual prepaid seats and a dedicated Usage
  page for tracking consumption.

  ### Fixes
  - Jira issue creation now paginates project and assignee dropdowns and links
    back to the originating finding.
  - The billing flow resolves your Stripe customer consistently across trials
    and resubscribes, and skips invoice previews for canceled subscriptions.
  - Large webhook payloads and oversized diagram traces no longer fail
    processing or abort PDF report generation.
</Update>

<Update label="May 2026" description="Hacktron Platform">
  ## Applications, a redesigned taint trace, and richer findings

  **Applications** — Group related repositories into an Application, give it
  context and rules, and target it directly from the scan wizard.

  **Redesigned taint trace** — The taint-trace and call-graph views have a new
  design with smooth collapse animations, making it easier to follow how a
  vulnerability flows through your code.

  **Finding history** — Findings now carry a visible event timeline with a
  deduplicated lineage trail, so you can see how each finding evolved across
  scans.

  ### Fixes
  - PR comments are now placed correctly on dotfiles and large-diff files.
  - The "Last scan" dashboard card is scoped to white-box scans.
  - Trial banners no longer clip the bottom of the page, and the
    trial-limit-reached call to action now ends the trial as expected.
</Update>
