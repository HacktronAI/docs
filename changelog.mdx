<Update label="June 2026" description="Hacktron Platform">
  ## GitLab reviews, self-hosted GitHub, and triage in your PRs

  **GitLab merge request reviews** — Comment `@hacktronai review` on a merge
  request to get a findings summary and inline review, on par with GitHub.

  **GitHub Enterprise Server** — Connect a self-hosted GitHub Enterprise
  instance and run white-box scans and PR reviews against it.

  **Triage in the conversation** — Triage decisions now appear directly in the
  GitHub PR or GitLab MR thread, so the discussion stays in one place.

  **Applications in the scan wizard** — Target an Application directly when you
  start a scan.

  **Pick a commit for your pentest** — Choose a specific tag or commit when you
  start a pentest.

  **Pooled, organization-wide billing** — Code Review billing is now shared
  across your organization, with annual seats and a new Usage page to track
  consumption.
</Update>

<Update label="May 2026" description="Hacktron Platform">
  ## Applications, a redesigned taint trace, and finding history

  **Applications** — Group related repositories into an Application and give it
  shared context and rules, so reviews understand how your code fits together.

  **Redesigned taint trace** — The taint-trace and call-graph views have been
  rebuilt so it's easier to follow how a vulnerability moves through your code.

  **Finding history** — Each finding now shows a timeline of how it changed
  across scans, with duplicates linked into a single lineage.

  **On-demand PR reviews** — Comment `@hacktron review` on a pull request to
  have Hacktron review it on the spot.
</Update>

<Update label="April 2026" description="Hacktron Platform">
  ## Public API, Code Review, and richer findings

  **Public REST API** — Programmatic access to your scans, findings, and
  repositories with API keys, plus interactive API docs.

  **Code Review** — A dedicated Code Review page with scan-scoped findings, and
  PR comments that link straight back to the finding in Hacktron.

  **Richer finding details** — Affected code is now syntax-highlighted and
  vulnerabilities include a visual trace diagram, in both the finding panel and
  the PDF report.

  **Context documents** — Upload per-repository context so reviews understand
  your codebase and conventions.

  **More flexible pentests** — Start a pentest from a public repository URL or
  an uploaded archive, with clearer cost estimation before you commit.

  **Self-serve billing and integrations** — Cancel or reactivate a subscription
  and buy credits in-app; finding and scan links unfurl in Slack with live
  status and triage; and you can create Linear tickets from findings.
</Update>
