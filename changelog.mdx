<Update label="June 11, 2026" tags={["Integrations", "Code Review"]}>
  ## GitLab onboarding and scanning reach GitHub parity

  **Automatic GitLab merge-request scanning** — Connect a GitLab project and
  merge-request scans turn on automatically, matching how GitHub repositories
  behave. You can still disable them per project.

  **GitLab in onboarding and trials** — Connect GitLab during signup or a trial
  and it works just like GitHub, with the same guided setup.

  **[<u>Set up GitLab →</u>](/platform/repositories/gitlab)**
</Update>

<Update label="June 10, 2026" tags={["Code Review", "Billing", "Integrations"]}>
  ## Fix from Slack, no-card trials, and self-closing findings

  **Fix with AI in Slack** — Finding notifications in Slack now include a "Fix
  with AI" action with one-click deep links into Cursor or Claude, so you can go
  from a Slack alert straight to fixing the issue in your editor.

  **No-card free trials** — Start a free trial without entering a credit card —
  no payment details until you decide to subscribe.

  **Findings close themselves on abandoned PRs** — When a pull request or merge
  request is closed without being merged, its findings move to a new "Closed"
  state, and reopening the PR restores them. Findings you've already triaged are
  left untouched.

  **Unsupported-language notices** — Cost estimation now warns you when a
  repository is mostly written in a language Hacktron can't yet scan, instead of
  failing without explanation.

  **Clearer GitLab connection setup** — The Connect GitLab dialog walks you
  through GitLab's group Service Accounts flow step by step, and adds a
  GitLab.com / Self-hosted toggle that mirrors the GitHub Enterprise setup.

  **[<u>Connect Slack →</u>](/platform/communication-apps/slack)** · **[<u>Start a free trial →</u>](/code-review/billing)**
</Update>

<Update label="June 7, 2026" tags={["Self-Hosting", "Integrations", "Code Review", "Whitebox"]}>
  ## GitHub Enterprise Server, GitLab MR feedback, and commit-level scans

  **GitHub Enterprise Server** — Connect self-hosted GitHub Enterprise Server
  instances for white-box scans and PR reviews, with multi-host management so
  several Enterprise hosts run alongside github.com.

  **GitLab merge-request feedback at parity with GitHub** — MR comments now
  carry severity badges, collapsible proof-of-concept, trace diagrams, and a
  "Fix with AI" block. Trigger a review with `@hacktronai review` on a merge
  request, and triage activity mirrors across the web app, Slack, and the MR
  thread.

  **Whitebox scans by tag or commit** — When picking a repository, target a
  specific tag or an exact commit, not just a branch.

  **Richer Jira ticketing** — Search across large Jira projects and assignee
  lists when filing a ticket, and issues created from findings link back to the
  original finding.

  **[<u>Set up GitHub Enterprise Server →</u>](/platform/repositories/github-enterprise-server)** · **[<u>See how reviews work →</u>](/code-review/integrations/github-gitlab)** · **[<u>Set up Jira →</u>](/platform/project-management/jira)**
</Update>

<Update label="June 3, 2026" tags={["Billing", "Code Review", "Integrations"]}>
  ## Org-pooled billing, redesigned navigation, and on-demand reviews

  **Org-pooled billing, annual seats, and a Usage page** — Code Review limits are
  now shared across your whole organization instead of being capped per seat, you
  can buy developer seats on an annual prepaid plan, and a new owner-only Usage
  page shows your usage and any overage for the period.

  **Redesigned sidebar navigation** — Cleaner, collapsible sections that remember
  their open and closed state between visits, with account actions in the
  sidebar header.

  **On-demand PR reviews** — Comment `@hacktron review` on a pull request to have
  Hacktron review it on the spot, even on drafts or external-contributor PRs.

  **Request GitHub access without being an org admin** — If you don't own the
  GitHub organization, clicking Connect now sends an installation request to
  your admin and shows it as pending; once approved, the integration appears
  automatically.

  **Smoother Jira setup** — A cleaner Jira configuration and per-ticket dialog,
  with sensible defaults already turned on for new installs.

  **Slack Connect onboarding** — Sign up with a work email and a Slack Connect
  step lets you set up notifications right from the start.

  **[<u>See billing and plans →</u>](/code-review/billing)** · **[<u>Set up Code Review →</u>](/code-review/setup)**
</Update>

<Update label="June 1, 2026" tags={["Whitebox", "Code Review"]}>
  ## Whitebox naming and redesigned finding views

  **"Whitebox" terminology** — Scans previously labeled "Pentest" are now called
  "Whitebox" everywhere, with "Whitebox Scans" and "PR Reviews" clearly
  separated so it's easy to tell full code scans from pull-request reviews.

  **Redesigned vulnerability views** — The taint-trace and call-graph views were
  rebuilt with a new visual design, opening straight into the trace with smooth
  collapse animations, and the full-page finding view now has the same actions
  as the sidebar.

  **Finding history** — Each finding now shows a timeline of how it changed
  across scans.

  **[<u>Explore Whitebox scans →</u>](/white-box-pentest/overview)**
</Update>

<Update label="May 26, 2026" tags={["Code Review"]}>
  ## Smarter duplicate detection

  **Cross-finding duplicate detection** — Hacktron now recognizes when a new
  finding duplicates one it already reported and links them automatically, so PR
  comments point back to the original finding instead of repeating it.
</Update>

<Update label="May 19, 2026" tags={["Dashboard", "Code Review", "Integrations"]}>
