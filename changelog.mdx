<Update label="June 11, 2026" tags={["GitLab", "Onboarding"]}>
  ## GitLab onboarding and scanning reach GitHub parity

  **Automatic GitLab merge-request scanning** — Connect a GitLab project and
  merge-request scans turn on automatically, matching how GitHub repositories
  behave. You can still disable them per project.

  **GitLab recognized in trials and onboarding** — Trial prompts, the "all set"
  onboarding screens, and trial banners now treat a connected GitLab org the
  same way they treat GitHub.

  **Clearer GitLab connection setup** — The Connect GitLab modal walks you
  through GitLab's group Service Accounts flow with corrected steps, and adds a
  GitLab.com / Self-hosted toggle that mirrors the GitHub Enterprise setup.

  **PR review nudge after cost estimation** — If you reach cost estimation and
  close it without enabling PR scans, a one-time prompt on the repositories page
  invites you to turn on PR review.
</Update>

<Update label="June 10, 2026" tags={["Slack", "Findings", "Billing"]}>
  ## Fix from Slack, no-card trials, and self-closing findings

  **Fix with AI in Slack** — Finding notifications in Slack now include a "Fix
  with AI" action with one-click deep links into Cursor or Claude, so you can go
  from a Slack alert straight to fixing the issue in your editor.

  **No-card free trials** — Start a free trial without entering a credit card,
  and brand-new organizations that have never subscribed now see the "Start
  Trial" button on the billing page.

  **Findings close themselves on abandoned PRs** — When a pull request or merge
  request is closed without being merged, its findings move to a new "Closed"
  state, and reopening the PR restores them. Findings you've already triaged are
  left untouched.

  **Unsupported-language notices** — Cost estimation now tells you when a
  repository is mostly written in a language Hacktron can't yet trace, with
  guidance to reach out.
</Update>

<Update label="June 7, 2026" tags={["GitHub Enterprise", "GitLab", "Jira"]}>
  ## GitHub Enterprise Server, GitLab MR feedback, and richer Jira

  **GitHub Enterprise Server** — Connect self-hosted GitHub Enterprise Server
  instances for white-box scans and PR reviews, with multi-host management so
  several Enterprise hosts run alongside github.com.

  **GitLab merge-request feedback at parity with GitHub** — MR comments now
  carry severity badges, collapsible proof-of-concept, trace diagrams, and a
  "Fix with AI" block. Trigger a review with `@hacktronai review` on a merge
  request, and triage activity mirrors across the web app, Slack, and the MR
  thread.

  **Richer Jira ticketing** — Project and assignee pickers now support
  server-side search and pagination, and issues created from findings link back
  to the originating finding.
</Update>

<Update label="June 4, 2026" tags={["Billing", "GitLab", "Whitebox"]}>
  ## Org-pooled billing, GitLab MR reviews, and commit-level scans

  **Org-pooled Code Review billing and a Usage page** — PR review limits are now
  pooled across your whole organization rather than capped per seat, with
  open-source contributions tracked separately. A new owner-only Usage page
  shows consumption for the billing period and an estimated cost for any
  overage.

  **Annual prepaid seats** — Purchase developer seats on an annual prepaid plan
  for a simpler commitment.

  **GitLab merge-request comments and triage** — GitLab merge-request comments
  and finding triage reach parity with GitHub, so you can review and act on
  findings without leaving GitLab.

  **Whitebox scans by tag or commit** — When picking a repository, target a
  specific tag or an exact commit, not just a branch, with live validation.

  **Slack Connect onboarding** — Users signing up with a work email now see a
  Slack Connect step during onboarding to wire up notifications from the start.
</Update>

<Update label="June 2, 2026" tags={["Applications", "Findings", "Integrations"]}>
  ## Applications in the scan wizard and smarter duplicate detection

  **Applications in the scan wizard** — Target an Application directly from the
  scan wizard to scan all of its repositories at once, with the selected repos
  collapsing into a single row.

  **Cross-finding duplicate detection** — Hacktron now recognizes when a new
  finding duplicates one it already reported and links them automatically, with
  lineage annotations and canonical-finding references in PR comments.

  **Request GitHub access without being an org admin** — If you don't own the
  GitHub organization, clicking Connect now sends an installation request to
  your admin and shows it as pending; once approved, the integration appears
  automatically.

  **Polished Jira integration setup** — The Jira configuration and per-ticket
  modals are cleaner and more reliable, with auto-create defaults on for fresh
  installs and required fields enforced before dismissal.
</Update>

<Update label="June 1, 2026" tags={["Whitebox", "Applications", "Findings"]}>
  ## A redesigned platform: Applications, new finding views, and on-demand reviews

  **Applications** — Group related repositories into a named Application, give it
  context and rules, and manage its repositories and configuration from a
  dedicated detail page.

  **"Whitebox" terminology** — Scans previously labeled "Pentest" are now called
  "Whitebox" everywhere, with "Whitebox Scans" and "PR Reviews" clearly
  separated so it's easy to tell full code scans from pull-request reviews.

  **Redesigned sidebar navigation** — Cleaner, collapsible sections that remember
  their open and closed state between visits, with account actions in the
  sidebar header.

  **Redesigned vulnerability views** — The taint-trace and call-graph views open
  straight into the trace with an inline Source / Propagation / Sink legend and
  smooth collapse animations, and the full-page finding view now has the same
  actions as the sidebar.

  **Finding history** — Each finding now shows a timeline of how it changed
  across scans, with duplicates linked into a single lineage.

  **On-demand PR reviews** — Comment `@hacktron review` on a pull request to have
  Hacktron review it on the spot, even on drafts or external-contributor PRs.

  **Sharper GitHub PR comments** — Pull-request comments now use crisp severity
  badges in place of emoji, with a hardened "Fix with AI" prompt that reproduces
  the issue, fixes the root cause, and adds a regression test.
</Update>

<Update label="April 2026" description="Hacktron Platform">
  ## Public API, Code Review, and richer findings

  **Public REST API** — Programmatic access to your scans, findings, and
  repositories with API keys, plus interactive API docs.

  **Code Review** — A dedicated Code Review page with scan-scoped findings, and
  PR comments that link straight back to the finding in Hacktron.

  **Richer finding details** — Affected code is now syntax-highlighted and
  vulnerabilities include a visual trace diagram, in both the finding panel and
  the PDF report.

  **Context documents** — Upload per-repository context so reviews understand
  your codebase and conventions.

  **More flexible pentests** — Start a pentest from a public repository URL or
  an uploaded archive, with clearer cost estimation before you commit.

  **Self-serve billing and integrations** — Cancel or reactivate a subscription
  and buy credits in-app; finding and scan links unfurl in Slack with live
  status and triage; and you can create Linear tickets from findings.
</Update>

<Update label="March 2026" description="Hacktron Platform">
  ## Cost estimation, live scan progress, and richer findings

  **Cost estimation before you scan** — See an estimated cost based on dynamic
  credit pricing before you start a scan or pentest.

  **Live scan progress** — Scans now stream their progress in real time, so you
  can watch findings arrive as they're discovered.

  **Triage from the web** — Dismiss findings with a reason and leave feedback in
  an activity thread, directly in the app.

  **Richer finding details** — Finding descriptions, dismiss reasons, and
  activity comments now render full Markdown, with one-click copy of the
  Markdown report.

  **Downloadable pentest reports** — Generate a PDF report of your pentest
  findings straight from the browser.
</Update>
