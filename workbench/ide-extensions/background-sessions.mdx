---
title: "Background Sessions"
description: "Run Hacktron security workflows in the IDE extension without blocking your editor"
---

Background sessions let you run Hacktron workflows in separate sessions while you keep working in your editor.

This is useful for longer scans and investigations that should continue running without interrupting the rest of your workflow.

## Why use background sessions

- Keep scans running while you continue editing code
- Separate different investigations into their own sessions
- Stay in your editor instead of switching back to the terminal

## How it fits with Workbench

Hacktron Workbench uses the same account across the CLI and IDE extensions, so you can move between the editor and terminal as needed.

If you want to continue related CLI work later, see [Sessions](/workbench/cli/sessions).
