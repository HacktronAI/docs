**Context** is where you give Hacktron the background it needs to review your code
well. Upload architecture notes and security policies, group related repositories
into applications, and let Hacktron build a threat model for each so reviews and
pentests reflect how your systems actually work.

Open **Context** from the sidebar. The page has two tabs:

- **Repositories** — every connected repository, each with its own context
  documents and threat model.
- **Applications** — named groups of related repositories that share context.

<img src="/images/context_overview.png" alt="The Context page showing the Repositories and Applications tabs" />

## Why context matters

Hacktron reviews your code more accurately when it understands your system. Context
documents and threat models tell the agent what your critical assets are, which
risks you have already accepted, and which findings you treat as false positives —
so it focuses on what matters to you.

This context feeds two places:

- **PR reviews** — Markdown context documents and the repository's threat model are
  used as shared context on every pull request review.
- **White-box pentests** — context documents and threat models are passed to the
  pentest so the engine starts with knowledge of your architecture.

## Context documents

A context document is a file you upload to a repository or application to inform
Hacktron's reviews. Supported formats:

| Format | Extensions | How it is used |
| --- | --- | --- |
| Markdown | `.md`, `.markdown` | Included in scans and the threat model. |
| Text | `.txt` | Treated as Markdown. |
| PDF | `.pdf` | Informs the threat model only. |

<Info>
  Good context documents include architecture overviews, data-flow notes, security
  policies, compliance requirements, API specs, and prior pentest reports.
</Info>

### Add a context document

<Steps>
  <Step title="Open Context and select a target">
    From the **Context** page, select **Add Context**. In the dialog, choose one or
    more repositories or applications the document applies to.

    <img src="/images/context_add_document.png" alt="The Add context document dialog" />
  </Step>
  <Step title="Upload a file or write one">
    Drag and drop a file, click to upload, or choose **Create a document** to write
    Markdown directly in the dialog. You can stage several files at once.

    Only `.md`, `.markdown`, `.txt`, and `.pdf` files are accepted. Unsupported
    files are skipped with a notice.
  </Step>
  <Step title="Add the document">
    Select **Add**. The document appears under each repository or application you
    selected and is queued to be folded into the threat model.
  </Step>
</Steps>

### View, edit, and remove documents

Open a repository or application from the **Context** page to see its documents in
the left sidebar.

- **View** — select a document to read it. Markdown documents show an outline; PDFs
  can be downloaded.
- **Edit** — select **Edit** on a Markdown document to update it inline, then save.
- **History** — switch to the **History** tab to see every revision, including
  automatic threat-model syncs.
- **Remove** — delete a document when it no longer applies. Its revision history is
  retained, and the threat model is updated to stop relying on it.

### Sync status

Each document shows whether it has been folded into the threat model:

| Status | Meaning |
| --- | --- |
| **Synced** | The document is included in the threat model. |
| **Pending** | A threat model sync is running and will include it shortly. |
| **Not synced** | The document is uploaded but not yet folded in. Run a sync to include it. |
