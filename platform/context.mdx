Context is where you give Hacktron the background it needs to review your code
well. Upload documents, group related repositories into applications, and let
Hacktron build a threat model for each.

Open **Context** from the sidebar. It has two tabs:

- **Repositories**: every connected repository, with its own documents and threat model.
- **Applications**: named groups of related repositories that share context.

<img src="/images/context_overview.png" alt="The Context page with the Repositories and Applications tabs" />

## Context documents

A context document is a file you upload to help Hacktron understand a repository or
application. Good examples are architecture notes, security policies, and prior
pentest reports.

Supported formats:

- **Markdown** (`.md`, `.txt`): used in scans and the threat model.
- **PDF** (`.pdf`): used in the threat model only.

### Add a document

<Steps>
  <Step title="Select Add Context">
    On the **Context** page, select **Add Context** and choose the repositories or
    applications the document applies to.

    <img src="/images/context_add_document.png" alt="The Add context document dialog" />
  </Step>
  <Step title="Upload or write">
    Drag in a file, or choose **Create a document** to write Markdown directly.
  </Step>
  <Step title="Add">
    Select **Add**. The document is saved and queued for the threat model.
  </Step>
</Steps>

### Manage documents

Open a repository or application to see its documents in the left sidebar. From
there you can view a document, download a PDF, edit Markdown inline, see past
versions in the **History** tab, or remove a document you no longer need.

Each document shows whether it is in the threat model yet: **Synced**, **Pending**,
or **Not synced**.

<img src="/images/context_document_status.png" alt="Context documents with Synced, Pending, and Not synced status" />
