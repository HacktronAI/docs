---
title: "Threat models"
description: "How Hacktron builds a threat model for each repository and application, keeps it up to date, and uses it to improve scans and PR reviews."
---

A threat model is a living document Hacktron keeps for each repository. It tracks
what matters to your security review, such as critical assets, accepted risks, and
recurring false positives, so reviews get better over time.

There are two kinds:

- **Repository threat model**: one per repository.
- **Application threat model**: one per application, built from its repositories.

<img src="/images/context_threat_model.png" alt="A repository threat model on the Context page" />

## Repository threat models

Hacktron creates a repository's threat model automatically, the first time a pull
request is reviewed or as soon as you upload context documents for it. After that
it keeps learning from how you triage findings.

When you add or change documents, the threat model needs to sync to include them.
This usually happens automatically. To sync now, open the repository and select
**Sync pending**.

<img src="/images/context_sync_pending.png" alt="Syncing pending documents into the threat model" />

## Application threat models

An application threat model gives Hacktron one view of a system that spans several
repositories. Hacktron builds it from each repository's threat model plus any
documents you upload at the application level.

To create one, open the application and select **Generate**. Hacktron creates any
missing repository threat models first, then builds the application threat model in
the background. After things change, select **Regenerate** to rebuild it.

<img src="/images/context_application_threat_model.png" alt="Generating an application threat model" />

## How they are used

- **PR reviews**: the repository threat model is shared context on every review.
- **White-box pentests**: the threat model is passed to the pentest so it starts
  with knowledge of your system.
