---
title: "Applications"
description: "Group related repositories into an application so Hacktron shares context across them and can build one threat model for the whole system."
---

An application is a named group of repositories from the same codebase. Hacktron
uses it as shared context for more accurate scans and PR reviews across your repos,
and it can build one threat model for the whole application.

Use applications when a single system spans several repositories, such as a
frontend, an API, and a shared library.

<img src="/images/context_applications_tab.png" alt="The Applications tab on the Context page" />

## Prerequisites

- At least one connected repository.
- You must be an organization admin to create, edit, or delete applications.

## Create an application

<Steps>
  <Step title="Open the Applications tab">
    On the **Context** page, select the **Applications** tab, then select **New
    application**.

    <img src="/images/context_new_application.png" alt="Creating a new application" />
  </Step>
  <Step title="Name it and add repositories">
    Enter a **Name**, then select the repositories that make up the application. A
    repository can belong to only one application at a time.
  </Step>
  <Step title="Create">
    Select **Create**. The application appears on the **Applications** tab.
  </Step>
</Steps>

## Edit or delete

Open an application to see its threat model, member repositories, and any
application-level documents in the left sidebar.

Select **Edit Application** to change its name or repositories. Removing a
repository detaches it but keeps it available on its own. To delete the
application, select **Edit Application**, then **Delete application**.

<Warning>
  Deleting an application is permanent. Its repositories are detached and stay
  available individually, and any application-level documents move to their
  repositories.
</Warning>
