The Hacktron Platform is the shared workspace for setting up Hacktron across your organization.

Use it to connect your tools, choose which repositories Hacktron should watch, manage who has access, and run larger workflows like pentests.

## How teams usually get started

Most teams begin by connecting GitHub in **Integrations**, then heading to **Repositories** to decide where Hacktron should run. From there, **People** and **Billing** are where admins shape access and budgets for the rest of the organization. Once that setup is in place, the Platform becomes the shared place to monitor rollout and run larger workflows like pentests.

## What each area is for

Each area has a clear job:

- **Integrations** connects GitHub and other supporting tools
- **Repositories** decides where Hacktron should actually run
- **People** controls who has access
- **Billing** manages seats, credits, and shared budgets
- **Pentests** is for larger, scoped assessments beyond day-to-day review

## Platform vs Workbench

The Platform is for shared, organization-level setup and management.

Hacktron Workbench is for individual usage in the CLI or IDE. Use Workbench when you want to work directly in code. Use the Platform when you need team-wide setup, access control, billing, and visibility.
