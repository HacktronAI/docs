Hacktron Code Reviews lets your team review pull requests from the Platform.

Connect GitHub, choose the repositories you want covered, and Hacktron will review qualifying pull requests. When code reviews are enabled for a repository, Hacktron leaves comments directly on the PR.

<Card img="/images/platform-repositories-dark.png">
  Use the **Repositories** page to choose where Hacktron should run and to confirm that your repositories are synced correctly.
</Card>

## What enabling code reviews does

After GitHub is connected, enable coverage from the **Repositories** page. If a repository is enabled and the PR qualifies for coverage, Hacktron reviews the pull request and comments on the PR.

## Setup

1. Connect GitHub in **Integrations**
2. Open **Repositories** and turn on the repositories you want covered
3. Let your team keep working in PRs as usual
4. Review coverage and findings from the dashboard

You do not need to manually invite every developer before getting started. Hacktron can discover developers from repository activity as reviews start running.

## If a PR is not being reviewed

The first things to check are:

- GitHub is connected in **Integrations**
- The repository is enabled in **Repositories**
- The organization still has review capacity available

If someone is not covered, Hacktron does not silently pretend the review ran. Instead, the product behavior is to leave a PR comment indicating that a seat is not assigned.

## During the trial

For new organizations:

- The free trial lasts **14 days**
- The trial begins when you choose to start it from the **Billing** page
- During the trial, seats auto-assign as developers open PRs
